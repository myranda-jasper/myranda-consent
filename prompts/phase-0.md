# Phase 0 — Prompt

> Saved verbatim. This is the prompt that kicked off the project.

---

You're building a new standalone, open-source hackathon project called "Myranda
Consent" in a fresh folder. Standing rules: public GitHub repo named
myranda-consent, commit in small steps and push after each, keep a PLAN.md and a
/prompts folder with every prompt I give you, do NOT touch my other project (the
On-device web agent folder), explain in plain English, and tell me exactly when
I must act. I am not deeply technical.

Phase 0:
1. Scaffold a Next.js app (App Router) with TypeScript and Tailwind, named
   myranda-consent.
2. Initialise git and create a PUBLIC GitHub repo. Use the gh CLI if available;
   if not, walk me through creating the repo on github.com and adding the
   remote.
3. Build a simple landing page: title "Myranda Consent", subtitle "Your data.
   Your keys. Your consent, on the record." Nothing else yet.
4. Deploy it to Vercel now so we have a live URL. Walk me through the Vercel
   login and deploy (CLI or connecting the repo in the Vercel dashboard,
   whichever is simpler for me).
5. Create PLAN.md outlining the four phases, and a /prompts folder. Save this
   prompt as prompts/phase-0.md.

Commit in small steps. When done, give me the live Vercel URL and the GitHub
repo URL.
